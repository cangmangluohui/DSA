# 07 动态规划 (背包、状态转移) 

## 核心思想
将复杂问题分解为重叠子问题，通过记录子问题的解来避免重复计算，通常自底向上推导。
- 时间复杂度：取决于**状态数量**与**状态转移**开销（如 0-1 背包为 O(N * V)）
- 空间复杂度：O(V) (以 0-1 背包一维滚动数组优化为例)

## 核心避坑指南
1. **状态定义与初始化**：必须明确 `dp[i][j]` 的确切物理含义（如：前 i 个物品放入容量为 j 的背包的最大价值）。特别注意边界条件，如 `dp[0][j]` 或 `dp[i][0]` 的初始化，0-1 背包求最大值时通常初始化为 0，但求最小值或恰好装满问题时需初始化为无穷大/负无穷。
2. **遍历顺序与空间优化**：在使用一维数组优化空间时，0-1 背包（每个物品最多选一次）内层容量必须**逆序**遍历，防止同一物品被重复选取；完全背包（物品无限）则**正序**遍历。
3. **状态转移方程**：严格区分“选”与“不选”的逻辑。0-1 背包的核心转移为 `dp[j] = max(dp[j], dp[j - weight] + value)`，务必确保 `j >= weight` 时才进行状态更新，防止数组越界。
