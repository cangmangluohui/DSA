# 06 贪心算法 

## 1. 核心思想与适用条件

### 核心思想
- **局部最优推导全局最优**：不从整体最优上加以考虑，所做的选择只是在某种意义上的局部最优解。
- **不可回退**：一旦做出选择，在当前步骤就不会再更改（与动态规划保留所有可能性的状态转移不同）。

### 适用条件（两大核心性质）
1. **贪心选择性质**：全局最优解可以通过一系列局部最优的选择来达到。
2. **最优子结构**：一个问题的最优解包含其子问题的最优解。

## 2. 贪心策略的证明方法

> ⚠️ **核心警告**：无固定代码模板！难点不在于写代码，而在于**证明贪心策略的正确性**。如果无法证明，或者能轻易构造出反例，请果断放弃贪心，改用动态规划 (DP)。

### 常用证明方法：
1. **反证法**：假设不是全局最优解，通过逻辑推导得出矛盾，从而证明贪心解即为最优解。
2. **数学归纳法**：证明边界情况（如 $n=1$）下贪心成立；假设前 $k$ 步贪心是最优的，证明第 $k+1$ 步贪心依然能保持最优（一般不用）。
3. **交换论证法**：假设存在一个最优解，通过“交换”其中的某些步骤，将其替换为贪心解，证明替换后的解不会变差。

## 3. 经典模型与贪心策略

| 模型分类 | 贪心策略 | 经典例题 | 备注 |
| :--- | :--- | :--- | :--- |
| **区间调度** | 按结束时间从小到大排序，尽量选结束早的 | 活动选择问题、无重叠区间 | 结束越早，留给后面的时间越多 |
| **区间覆盖** | 起点相同选最长，或按左端点排序选右端点最远的 | 喷水装置、最少区间覆盖 | 注意边界处理与多遍扫描 |
| **霍夫曼编码** | 每次选两个最小的合并 | 合并果子、霍夫曼树 | 使用优先队列（小顶堆）维护 |
| **跳跃游戏** | 每次维护当前能到达的最远边界 | Jump Game I & II | 贪心维护右边界，判断能否到达终点 |
| **字典序/排列** | 比较 $a+b$ 和 $b+a$ 的大小 | 拼接最小/最大数 | 自定义排序规则，需满足严格弱序 |

| **分配问题** | 排序后双指针，或贪心匹配 | 分发饼干、买卖股票 | 尽量满足小需求，或维护状态DP |

## 4. 核心避坑指南

1. **不要凭直觉贪心**：一定要尝试构造反例！如果构造出了反例，立刻转 DP。
2. **排序规则要严谨**：自定义 `sort` 比较函数时，必须满足**严格弱序**（不能出现 $a>b$ 且 $b>a$ 同时成立的情况），否则会导致程序崩溃（如 C++ 的 `std::sort` 越界或死循环）。
3. **边界条件与极端情况**：贪心往往伴随着排序和遍历，务必注意数组越界、空集、单个元素、全相同元素等极端情况。
4. **警惕“伪贪心”与状态依赖**：当前选择改变后续状态空间（例如带手续费的股票买卖问题），此时贪心失效，必须使用动态规划记录状态。
